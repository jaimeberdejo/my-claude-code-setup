# ADR-008: The enforcement and UAT ledgers are removed (supersedes ADR-004)

Date: 2026-07-16
Decision: `docs/ENFORCEMENT.md` and `docs/UAT.md`, and their validators `scripts/lint-enforcement.sh` and `scripts/check-uat.sh` (with their suites), are removed in v2.15.0. **This supersedes ADR-004.** Bounded planner gap planning survives on its own terms — it is triggered by missing evidence, an evaluator NEEDS_WORK, an ownership violation, or an invalidated stale plan, none of which needed a ledger. Reinstating either ledger requires a fresh ADR and a producer, not a revert; `test-docs-invariants.sh` pins the removal so it cannot creep back silently.
Why: v2.14.0 shipped both validators with **no producer, no template, and no caller** — no skill, command, agent, hook, or CI step created or read either artifact, so ~470 lines were reachable only from their own fixtures. The plan promised the missing half ("shipped ENFORCEMENT.md/UAT.md templates") and it never shipped, so neither file existed in the toolkit or in any of three real consumer repos. Meanwhile `AUTHORING.md` graded them **Deterministic** and the README stated `check-uat.sh` "blocks a release" — a gate that no release path invoked. That inversion (a documented guarantee resting on unreachable code) is the precise failure this project's own guarantee table exists to prevent, and it is how both shipped fail-open: an all-skipped ledger reported "structure OK", and a FAILED acceptance item with no `Blocking:` field passed `--strict`. ADR-007's standard decides it — "these deferred systems are revisited only when a real consumer demonstrates the need" — and three of three real consumers demonstrated the opposite. The rejected alternative, **wiring `check-uat.sh --strict` into `release-check.sh`**, was rejected precisely because it looks like the fix: it would add a release gate that calls an inert validator against a file nothing creates, always exit 0, and mint a new deterministic-sounding row in the guarantee table — buying the appearance of enforcement at the cost of one more thing to maintain. Deletion is reversible from git history the day a real consumer produces the artifact; a green gate proving nothing is not.
