# ADR-009: STANDARD plan review is risk-proportionate (deterministic routing at the /phase gate)

Date: 2026-07-19
Decision: `/phase` routes the plan gate through a new deterministic `scripts/plan-review-route.sh` rather than a bare tier label plus model judgement. The router validates the persisted `docs/SPEC.md` `tier:` (invalid or absent → fail safe to STANDARD + full review), probes the plan's declared paths for high-stakes risk via the shared `.claude/lib/_high-stakes.sh`, checks plan freshness, and prints a visible decision block (Selected tier / Risk signals / Plan review / Reason / Override / Supervised). A **clear low-risk STANDARD** phase — no high-stakes path, plan fresh, no blocking `[NEEDS CLARIFICATION]` — takes deterministic checks only and dispatches **no** Evaluator PLAN_CHECK; risky STANDARD, DEEP, supervised, an invalid tier, a hard-stale plan, or a blocking clarification force the full independent PLAN_CHECK; a high-stakes path overrides a false or stale TINY. No new tier, no new agent, no persistent review-state machine; the router never ticks, grades, or emits a gradeable token.
Why: R4's tiers scaled ceremony but STANDARD was monolithic — every STANDARD phase paid for a full agent PLAN_CHECK even when the change was reversible and low-risk — and the persisted `tier:` was never validated, so a false or stale TINY silently bought less review (the exact gap the AUTHORING guarantee table admitted). A deterministic script cannot be talked out of the routing by a persuasive prompt, is mutation-testable, and keeps the decision inspectable and visible — the discipline this project already applies to every other gate. The rejected alternative, prose-only guidance in `phase.md`, cannot deterministically prevent silent under-classification and is hard to prove non-vacuous. The routing signal deliberately covers the tier value plus the plan's DECLARED paths (not the implementation, which does not exist at plan time), so the guarantee stays honest rather than over-claiming enforcement.
